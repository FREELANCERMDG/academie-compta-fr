# Logiciel : Quadra (Cegid Quadra / QuadraCOMPTA)

**Niveau :** Intermédiaire → Avancé · **Durée estimée :** 7–9 h (pratique) · **Prérequis :** Modules 4–11
**Type :** Formation logiciel — outil de production historique très répandu en cabinet.

---

## 🎯 Objectifs pédagogiques
- Saisir efficacement avec les **guides d'écriture** Quadra.
- Lettrer, rapprocher la banque, éditer grand livre/balance.
- Utiliser les modules de **révision** et d'**immobilisations**.

## 🧩 Compétences
- Saisie rapide par guides (TVA pré-paramétrée).
- Justification des comptes en révision.

---

## 1. À quoi sert Quadra
**QuadraCOMPTA** (suite Cegid Quadra) est un logiciel de **production comptable cabinet** : saisie, lettrage, rapprochement bancaire, états, **révision (QuadraEXPERT)**, **immobilisations**, **liasse fiscale**. Orienté productivité (saisie au clavier rapide, guides d'écriture, abonnements).

> Selon le cabinet : version installée ou hébergée. Les **menus exacts varient** selon la version — ici on décrit les **fonctions** (stables).

---

## 2. Repères d'interface
- **Dossier / société** + **exercice** ouverts (toujours vérifier avant de saisir).
- **Journaux** (codes : AC, VT, BQ, OD…) et **périodes**.
- **Plan comptable** du dossier.
- **Guides d'écriture** : modèles d'écritures pré-paramétrés (compte + TVA) pour saisir vite.

---

## 3. Workflow de production
1. **Ouvrir** le bon dossier + le bon exercice + la bonne période.
2. **Saisir** par journal, en s'appuyant sur les **guides d'écriture** (vérifier le compte et la TVA produits par le guide).
3. **Importer/saisir la banque**, puis **rapprochement bancaire**.
4. **Lettrer** 401/411 (auto + manuel pour les écarts).
5. **Éditer** grand livre / balance pour contrôle.
6. **Réviser** par cycle (QuadraEXPERT) : justifier les soldes.
7. **Immobilisations** : tenir le fichier, passer les dotations.
8. Préparer les éléments de **liasse** → validation EC.

---

## 4. Les guides d'écriture (clé de la productivité)
- Un guide = un **modèle** (ex. « Achat fournisseur TVA 20 % » → 607 / 44566 / 401).
- Gain de temps énorme **mais** : si le guide est mal choisi, la **TVA ou le compte sont faux** → toujours **vérifier le résultat**.
- Les **abonnements** automatisent les écritures récurrentes (loyer, leasing) : contrôler les montants.

## 5. Rapprochement & lettrage
- Rapprochement : pointer relevé ↔ 512 (cf. Module 6) ; virements internes via 580.
- Lettrage : auto par montant, manuel pour avoirs/écarts (cf. Module 7).

## 6. Erreurs fréquentes
- Mauvais **dossier/exercice/période** ouverts (saisie au mauvais endroit).
- **Guide d'écriture** inadapté (mauvaise TVA pré-paramétrée).
- Brouillard non validé (écritures « en attente » oubliées).
- Lettrage automatique sur des écarts.

## 7. Bonnes pratiques offshore
- **Vérifier le contexte** (dossier/exercice/période) avant chaque session.
- Contrôler le rendu des guides automatiques.
- Valider les brouillards, contrôler la balance après saisie.
- Travailler **par cycle** en révision et justifier chaque solde (feuille de travail).

## 🧪 Cas pratique 23.1 — Saisie par guide
Vous saisissez une facture d'achat de marchandises 1 200 € TTC via le guide « Achat TVA 20 % ». Le guide propose 607 / 44566 / 401. La facture est en réalité une **prestation de sous-traitance**. Que faites-vous ?

**✔️ Corrigé :** le **compte de charge est faux** (607 = marchandises). Il faut le corriger en **611 (sous-traitance)** ; la TVA 20 % et le 401 restent corrects. Leçon : un guide accélère mais **ne dispense pas de vérifier la nature** de la charge. *(HT 1 000 / 44566 200 / 401 1 200.)*
