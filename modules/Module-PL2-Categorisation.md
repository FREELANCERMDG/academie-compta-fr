## Ce que montre la vidéo
Comment créer une **règle d'automatisation** pour **catégoriser automatiquement** les transactions bancaires récurrentes.

## Les étapes clés
1. Aller dans **Saisie → Transactions bancaires** (les opérations « à traiter »).
2. Repérer une opération **récurrente** (ex. « ORANGE », « OVH », loyer, abonnement…).
3. Créer une **règle** : *si le libellé contient « … »* → **compte de contrepartie** + **taux de TVA** (+ axe analytique si besoin).
4. La règle s'applique **automatiquement** aux prochaines transactions identiques.
5. **Contrôler** régulièrement les règles existantes pour éviter les mauvaises imputations.

## À retenir (réflexes cabinet)
- Les règles font **gagner énormément de temps**, mais elles **ne remplacent pas le contrôle**.
- Objectif d'un dossier propre : **0 transaction « à traiter »** en fin de période.
- Une règle trop large = écritures fausses : préférez des conditions **précises**.
