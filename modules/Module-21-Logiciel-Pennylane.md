# MODULE 21 — Logiciel : Pennylane (production & pré-compta)

**Niveau :** Intermédiaire · **Durée estimée :** 6–8 h (pratique) · **Prérequis :** Modules 4–7
**Type :** Formation logiciel — plateforme très utilisée par les cabinets français modernes.

---

## 🎯 Objectifs pédagogiques
- Maîtriser le workflow de production Pennylane côté collaborateur.
- Rapprocher transactions bancaires ↔ factures, catégoriser, lettrer, préparer la TVA.
- Gérer les justificatifs et les pièces manquantes dans Pennylane.

## 🧩 Compétences à maîtriser
- Traiter une « to-do » bancaire de A à Z.
- Catégoriser sans erreur et joindre la pièce systématiquement.

---

## 1. À quoi sert Pennylane
Plateforme **tout-en-un en ligne** (cloud) : connexion bancaire automatique, collecte et **océrisation des factures**, saisie/catégorisation, **rapprochement**, lettrage, **TVA**, reporting et facturation. Le cabinet pilote la production ; le client dépose ses pièces et consulte ses tableaux de bord.

**Deux espaces :**
- **Espace cabinet** (production) : où travaille le collaborateur.
- **Espace client** : dépôt de pièces, connexion bancaire, validation.

---

## 2. Avant de saisir : vérifier le paramétrage du dossier
- Le **régime TVA** (franchise / réel simplifié / réel normal, débits ou encaissements) est correct.
- Les **comptes bancaires** sont bien connectés et synchronisés (pas d'interruption).
- Le **plan comptable** du dossier et les **règles d'automatisation** existantes.
- Les **journaux** (achats, ventes, banque, OD).

> 🇲🇬 Posture offshore : on ne modifie pas le paramétrage du dossier sans accord du chef de mission.

---

## 3. Le workflow de production (étape par étape)
1. **Collecte** : les factures arrivent (upload client, mail dédié, océrisation). Vérifier l'OCR (montant, TVA, date, fournisseur).
2. **Banque** : les transactions descendent automatiquement. Traiter la **« to-do »** transaction par transaction.
3. **Rapprochement** : pour chaque transaction, **associer la facture/justificatif** correspondant.
4. **Catégorisation** : si pas de facture, affecter le **bon compte** (charge/produit) avec la **bonne TVA**.
5. **Lettrage** : les paiements se lettrent avec les factures clients/fournisseurs.
6. **Pièces manquantes** : marquer/relancer ; en attente → laisser non rapproché **avec note**.
7. **TVA** : contrôler puis générer le brouillon de déclaration → **validation chef de mission**.

---

## 4. Catégorisation et TVA
- Toujours vérifier le **taux et le compte de TVA** proposés (ne pas faire confiance aveuglément à l'automatisation).
- TVA sur **assurance** = exonérée ; **VP** = non déductible ; **carburant** = règle spécifique (Module 5).
- Les **règles automatiques** (catégorisation récurrente) : utiles mais **à contrôler**, surtout les premières fois.

## 5. Justificatifs & GED
- **Chaque écriture doit avoir sa pièce** jointe (c'est la force de Pennylane).
- Ne jamais valider une transaction « par dépense » sans justificatif → utiliser une note + relance.
- Surveiller l'alerte **doublon** (facture importée 2 fois).

## 6. Erreurs fréquentes
- Valider une transaction sans pièce.
- Catégorisation automatique acceptée sans contrôle (mauvais compte).
- Doublon import + saisie.
- Oublier la TVA sur **encaissements** (prestations de services).
- Connexion bancaire expirée → transactions manquantes (à signaler).

## 7. Bonnes pratiques offshore
- Traiter la **to-do bancaire chronologiquement**, ne rien laisser « gris ».
- Joindre la pièce **avant** de valider.
- Documenter chaque transaction inhabituelle (note interne).
- Préparer la TVA quelques jours **avant** l'échéance ; ne jamais la valider/télétransmettre seul.

---

## ✅ Checklist Pennylane (clôture mensuelle d'un dossier)
- [ ] Connexion bancaire active, toutes les transactions descendues
- [ ] To-do bancaire à zéro (tout rapproché ou justifié)
- [ ] OCR des factures contrôlé
- [ ] Catégorisations vérifiées (compte + TVA)
- [ ] Justificatifs joints partout
- [ ] Doublons contrôlés
- [ ] Lettrage 401/411 fait
- [ ] Pièces manquantes relancées + notées
- [ ] Brouillon TVA contrôlé → transmis pour validation

## 🧪 Cas pratique 21.1 — Traiter une to-do bancaire
On a 5 transactions : (a) virement reçu client (facture présente), (b) prélèvement Orange (facture présente), (c) prélèvement assurance GAN (facture présente), (d) achat carburant station (ticket), (e) virement émis non identifié.

**✔️ Corrigé :**
- (a) rapprocher avec la facture client → lettré automatiquement.
- (b) rapprocher avec la facture Orange (6262 + TVA 20 %).
- (c) rapprocher avec GAN → **616, pas de TVA** (assurance exonérée).
- (d) carburant → compte 60622, **TVA partielle** selon véhicule ; joindre le ticket.
- (e) inconnu → **laisser non rapproché + note + relance** (équivaut au 471).
